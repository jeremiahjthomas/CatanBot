# SPEC — `imitationLearning` branch

## What this branch does

Builds the data pipeline for imitation learning from Colonist.io replays.
Replaces the planned PPO self-play approach with behavioral cloning on real human games,
operating natively in Colonist's coordinate system.

---

## Problem: Colonist.io API is Turnstile-protected

The `data-from-game-id` endpoint (used to load replay data) requires a
Cloudflare Turnstile token generated by the browser's JavaScript. Plain
HTTP requests always return 403 regardless of auth cookies.

**Solution:** Playwright with real Chrome + automation suppression.

## `colonist/capture.py`

### Key design decisions

**Why Playwright instead of requests?**
The `data-from-game-id` endpoint sends a `turnstileToken` header generated
by Cloudflare's JavaScript challenge. This token is time-limited and can
only be produced by a real browser. Headless Chromium fails because
Cloudflare detects `HeadlessChrome` in the `sec-ch-ua` header.

**Why `channel="chrome"` (real Chrome)?**
Colonist.io uses Google OAuth. Playwright's bundled Chromium is flagged
as "unsupported browser" by Google's login flow. Using the user's installed
Chrome avoids this.

**Why `--disable-blink-features=AutomationControlled`?**
Even with real Chrome, Playwright sets `navigator.webdriver = true` by
default. Colonist's frontend JS detects this and does not load the replay,
which means `data-from-game-id` is never called. This flag (plus an
`add_init_script` override) hides the automation flag.

### Flow

```
--login mode:
  launch_persistent_context(real Chrome profile)
  → navigate to colonist.io
  → user logs in manually
  → save context.cookies() to colonist_cookies.json

download mode:
  launch Chrome + inject saved cookies
  → page.on("response", capture data-from-game-id)
  → navigate to /replay?gameId=<id>&playerColor=<color>
  → wait up to 30s for 200 response
  → write JSON to replays/<game_id>.json
```

### Rate limiting

Colonist.io rate-limits the replay API. Consecutive downloads use a 1s
delay between games. If rate-limited, wait ~1 hour before retrying.

## `colonist/parse_replay.py`

Replays are event streams. Each event has:
- `input.deltaS` — time elapsed
- `stateChange` — a partial diff of game state (map, players, bank, dice, etc.)

The parser reconstructs cumulative game state by replaying the diff stream,
then extracts action records from `gameLogState` entries.

### Action types extracted

`dice_roll`, `build_settlement`, `build_city`, `build_road`,
`place_robber`, `steal`, `play_dev_card`, `end_turn`,
`receive_resources`, `discard`, `monopoly`, `trade`, `game_over`

### Output schema (per action record)

```json
{
  "event_idx": int,
  "turn": int,
  "player": int,           // Colonist color (1, 5, etc.)
  "player_name": str,
  "action": { "type": str, ... },
  "resources_before": [int],
  "resources_after": [int],
  "vp": int,
  "buildings": {
    "settlements": [str],  // Colonist vertex IDs
    "cities": [str],
    "roads": [str]         // Colonist edge IDs
  },
  "robber_tile": int
}
```

## Coordinate system

Colonist.io uses fixed integer IDs for vertices (0–53) and edges (0–71).
These are consistent across all games (board topology is fixed; only
tile resources/numbers vary per game).

The mapping file (`colonist_to_catan_mapping.json`) covers:
- All 54 vertices ✅
- 62 of 72 edges ✅ (10 missing; 7 appear in real games — future work)

## Next steps

1. **Feature encoder** — convert a parsed state record into a fixed-size
   float vector: tile layout (resource × number × 19), vertex ownership
   (54 × 3), edge ownership (72 × 2), per-player resources/VP/dev cards,
   robber tile, whose turn.

2. **Action encoder** — map `(action_type, vertex_or_edge_id)` to a flat
   action index. Action space size: ~150–200 (54 build targets +
   72 road targets + ~30 non-placement actions).

3. **Behavioral cloning loop** — cross-entropy loss on `(state_vec, action_idx)`
   pairs, filtering to the acting player's perspective only.

4. **Evaluation** — win rate against rule-based heuristics; top-k accuracy
   on held-out replays.
